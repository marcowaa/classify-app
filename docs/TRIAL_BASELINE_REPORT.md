## 1) ما هو منفذ فعليا (مثبت من الكود)

1. سياسة تحديد عمر التحويل طفل/ولي امر من الادمن.
- قراءة/حفظ agePolicy متوفر.
- التطبيق فعليا في age gate.

2. بدء Parent Trial مباشر من age gate.
- انشاء حساب تجريبي + token + uniqueCode + classification.

3. بدء Child Trial مباشر من age gate.
- انشاء child trial وتوجيه مسار الطفل.

4. ربط تلقائي للطفل بعد تسجيل/دخول ولي الامر عبر trialChildToken.
- endpoint مخصص للربط موجود.
- منطق الاستدعاء بعد auth موجود.

5. حفظ نية شراء الطفل (Purchase Intent) واستعادة الفاتورة بعد الربط.
- intent محفوظ local storage.
- redirect الى parent store مع trialIntent.

6. طلب ربط ولي الامر عند محاولة الطفل الشراء بدون ربط.
- اظهار share code + link + qr في child store.

7. اعدادات trial من الادمن.
- explorePromptPercent.
- purchaseIntentPromptEnabled.
- showSocialLoginButtons.
- firstProductDiscountEnabled/Percent.

8. خصم اول منتج (سياسة trial) مطبق في checkout server side.

9. حملات اعلانات موجهة + روابط promo للمتجر + attribution اساسي.

10. اصلاح logout ولي الامر للرجوع الى age gate.

## 2) النواقص الفعلية (غير مكتملة)

تنبيه:
- هذه البنود كانت فجوات عند لحظة baseline فقط.
- تم تتبع إغلاقها لاحقا عبر مراحل P1..P7 في الخطة الرئيسية.

1. محرك سياسات مركزي موحد (Policy Engine) يطبق القرار على كل الصفحات.
- القرار حاليا موزع بين صفحات متعددة.

2. مصفوفة صلاحيات رسمية موحدة لكل حالة حساب (Parent Trial/Child Trial/Linked/Full).
- لا توجد طبقة قرار واحدة مع جدول واضح قابل للصيانة.

3. نظام موحد لبطاقات شرح الاقسام بهدوء عبر كل المشروع.
- موجود جزئيا وليس كنظام شامل.

4. تطبيق موحد لعتبة 30% عبر جميع اقسام ولي الامر.
- التطبيق الحالي جزئي في تدفقات محددة.

5. مدير Upgrade Prompt مركزي يغطي كل الميزات المقيدة بنفس السلوك.
- يوجد تنفيذات محلية في شاشات معينة.

6. تنسيق كامل بين العروض/الخصومات/الاشعارات (calm/popup/strong) كمنظومة واحدة.
- توجد اجزاء جاهزة لكن ليست موحدة end-to-end.
