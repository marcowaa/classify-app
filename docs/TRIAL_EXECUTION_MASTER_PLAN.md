# الخطة التنفيذية الرئيسية - تجربة الطفل وولي الامر (Master Plan)

حالة الوثيقة: CLOSED (P8 DONE)
مالك التنفيذ: Copilot + صاحب المنتج
آخر تحديث: 2026-03-21

## قبل اي عمل (قاعدة تشغيل)

1. اقرأ هذا الملف بالكامل في بداية كل جلسة.
2. ابدأ دائما من قسم متابعة التنفيذ ثم نفذ المرحلة التالية فقط.
3. لا تقفز بين المراحل بدون اغلاق المرحلة السابقة بمعيار النجاح المكتوب.
4. اي تعديل خارج نطاق المرحلة الحالية يعتبر ممنوع.
5. بعد انتهاء كل مرحلة: حدّث حالة المرحلة، وما تم، وما لم يتم.

## هدف الخطة

بناء نظام تجربة تجريبية متكامل للطفل وولي الامر بدون ترقيع، مع:
- فصل العمر (طفل/ولي امر) من لوحة الادمن.
- تجربة تصفح تجريبية ذكية.
- ربط تلقائي بين الطفل وولي الامر عند التسجيل.
- استكمال شراء المنتج الذي بدأه الطفل بعد الربط.
- نظام عروض وخصومات وحملات موجهة عبر المتجر والاشعارات.

2. تعريف قواعد قرار موحدة للعمر (اقل = طفل، عند/اكثر = ولي امر).

معيار النجاح:

### مصطلحات ومعايير معتمدة

مصطلحات:
1. Parent Trial: حساب ولي امر تجريبي بصلاحيات محدودة.
2. Child Trial: حساب طفل تجريبي للعب وجمع النقاط.
3. Linked: حالة نجاح ربط الطفل بولي الامر.
4. Upgrade Prompt: نافذة تطلب التسجيل/الربط لحفظ البيانات واستكمال المزايا.
5. Purchase Intent: نية شراء محفوظة لاستئناف الفاتورة بعد الربط.

قاعدة قرار العمر:
1. age < parentThresholdAge => Child Trial.
2. age >= parentThresholdAge => Parent Trial.

اتفاقية التسمية:
1. المفاتيح العامة تبدأ بـ trial* للحالات التجريبية.
2. مفاتيح الربط تبدأ بـ trialChild* او parentLink* حسب المسار.
3. مفاتيح سياسة الادمن توضع داخل trialPolicy و agePolicy.
